# Бизнес-требования: "Журнал личных учебных достижений"

## 1. Введение

### 1.1. Цель документа
Документ описывает бизнес-требования к программному обеспечению "Журнал личных учебных достижений", предназначенному для систематизации, учета и анализа учебных достижений пользователя.

### 1.2. Область применения
Программа предназначена для:
- Студентов всех форм обучения
- Школьников, участвующих в олимпиадах и конкурсах
- Преподавателей, ведущих учет достижений учащихся
- Любых лиц, желающих систематизировать свои учебные успехи

### 1.3. Бизнес-цели
1. Автоматизация учета учебных достижений
2. Визуализация прогресса и статистики
3. Упрощение создания отчетов и портфолио
4. Обеспечение долгосрочного хранения информации о достижениях

## 2. Описание продукта

### 2.1. Общее описание
Десктоп-приложение на Python с графическим интерфейсом (Tkinter) для управления базой данных учебных достижений.

### 2.2. Функциональные возможности

#### 2.2.1. Управление достижениями
- **Добавление новых достижений** с указанием:
  - Названия (обязательное поле, минимум 3 символа)
  - Даты (в формате ГГГГ-ММ-ДД)
  - Типа (выбор из списка)
  - Уровня (локальный, региональный, национальный, международный)
  - Описания (опционально)
- **Редактирование существующих записей**
- **Удаление записей** с подтверждением
- **Просмотр детальной информации** о достижении

#### 2.2.2. Отображение и навигация
- **Вкладка "Добавить"** - форма ввода новых данных
- **Вкладка "Мои достижения"** - табличный список всех записей
- **Вкладка "Статистика"** - аналитика и графики
- **Вкладка "Поиск"** - расширенный поиск с фильтрами

#### 2.2.3. Экспорт данных
- **Экспорт в Word** (.docx) с форматированием
- **Экспорт в Excel** (.xlsx) для анализа
- **Экспорт в PDF** (текстовый формат)
- **Сохранение графиков** в форматах PNG, PDF, SVG

#### 2.2.4. Аналитика и отчетность
- **Статистика по типам** достижений
- **Статистика по уровням** (с круговой диаграммой)
- **Динамика по месяцам** (последние 12 месяцев)
- **Визуализация данных** через matplotlib

#### 2.2.5. Поиск и фильтрация
- Поиск по названию (частичное совпадение)
- Фильтрация по типу и уровню
- Поиск по диапазону дат
- Сброс параметров поиска

#### 2.2.6. Вспомогательные функции
- **Резервное копирование** базы данных
- **Контекстное меню** для быстрых действий
- **Валидация ввода** данных
- **Подсветка обязательных полей**

### 2.3. Технические требования
- **Платформа**: Windows, Linux, macOS
- **Интерфейс**: графический (Tkinter)
- **Хранение данных**: SQLite база данных
- **Конфигурация**: JSON файл с типами достижений

## 3. Пользовательские роли

### 3.1. Основной пользователь
- Добавляет, редактирует, удаляет свои достижения
- Просматривает статистику
- Экспортирует данные в различные форматы
- Выполняет поиск по своим записям

## 4. Бизнес-правила

### 4.1. Правила валидации
1. Название достижения должно содержать минимум 3 символа
2. Дата должна быть в формате ГГГГ-ММ-ДД
3. Дата не может быть будущей (может быть реализовано в будущем)
4. Все поля, кроме описания, обязательны для заполнения

### 4.2. Правила отображения
1. Список достижений сортируется по дате (сначала новые)
2. Длинные описания обрезаются в списке (97 символов + "...")
3. Статистика обновляется автоматически при изменении данных

### 4.3. Правила безопасности
1. Хеширование названий для дополнительной безопасности (демонстрационное)
2. Подтверждение при удалении записей
3. Резервное копирование по запросу пользователя

## 5. Требования к интерфейсу

### 5.1. Основные принципы
- **Интуитивность**: понятная навигация по вкладкам
- **Консистентность**: единый стиль элементов
- **Обратная связь**: сообщения об успешных операциях и ошибках
- **Доступность**: поддержка клавиатурных сокращений

### 5.2. Визуальный дизайн
- Современный плоский дизайн
- Зеленые акцентные цвета для основных кнопок
- Четкое разделение секций
- Поддержка темной темы (возможность расширения)

## 6. Требования к данным

### 6.1. Структура данных
```
Достижения:
- ID (автоинкремент)
- Название (текст)
- Дата (текст в формате ГГГГ-ММ-ДД)
- Тип (текст из предопределенного списка)
- Уровень (текст: локальный/региональный/национальный/международный)
- Описание (текст, необязательно)
- Created_at (метка времени создания)

Статистика:
- ID (автоинкремент)
- Тип (текст)
- Количество (целое число)
- Месяц (текст в формате ГГГГ-ММ)
```

### 6.2. Конфигурационные файлы
- `types.json` - список типов достижений (редактируемый)
- `достижения.db` - основная база данных SQLite

## 7. Требования к производительности

### 7.1. Скорость работы
- Загрузка списка из 1000 записей: не более 2 секунд
- Поиск с фильтрами: не более 1 секунды
- Генерация статистики: не более 0.5 секунды

### 7.2. Надежность
- Автоматическое создание резервных копий (возможность расширения)
- Восстановление после сбоев
- Проверка целостности базы данных

## 8. Интеграционные требования

### 8.1. Внешние зависимости
- Библиотеки для работы с документами (python-docx)
- Библиотеки для работы с Excel (pandas, openpyxl)
- Библиотеки для графиков (matplotlib)
- Компоненты календаря (tkcalendar)

### 8.2. Форматы экспорта
- Microsoft Word (.docx)
- Microsoft Excel (.xlsx)
- Текстовые файлы (.txt) с возможностью печати в PDF
- Изображения графиков (.png, .pdf, .svg)

## 9. Ограничения и зависимости

### 9.1. Технические ограничения
- Требуется Python 3.8+
- Необходимо 100 МБ свободного места
- Требуется поддержка графического интерфейса
- Для экспорта в Word требуется установленный шрифт поддержки Unicode

### 9.2. Бизнес-ограничения
- Бесплатное распространение
- Открытый исходный код
- Поддержка только одного пользователя на инстанс
- Локальное хранение данных
