# Product Backlog: Журнал личных учебных достижений

## Общее описание проекта
Приложение для учёта и систематизации личных учебных достижений с графическим интерфейсом на tkinter, базой данных SQLite и возможностью экспорта отчётов в формате Word.

---

## Эпик 1: Базовая архитектура и настройка проекта

### US-1: Создание базовой структуры проекта
**Как** пользователь-разработчик,  
**Я хочу** иметь чёткую структуру проекта,  
**Чтобы** обеспечить лёгкую поддержку и расширение приложения.

**Критерии приёмки:**
- [ ] Проект содержит только разрешённые библиотеки (tkinter, sqlite3, json, python-docx)
- [ ] Все файлы находятся в одной папке
- [ ] Отсутствуют сложные ООП конструкции (только один основной класс)
- [ ] Не используются регулярные выражения

**Оценка сложности:** 2

### US-2: Настройка конфигурационного файла типов
**Как** пользователь,  
**Я хочу** иметь возможность настраивать типы достижений через внешний файл,  
**Чтобы** адаптировать приложение под свои нужды.

**Критерии приёмки:**
- [ ] Существует файл `types.json` в кодировке UTF-8
- [ ] Приложение корректно загружает типы из JSON
- [ ] При отсутствии файла используются типы по умолчанию
- [ ] Поддерживается динамическое обновление типов

**Оценка сложности:** 3
