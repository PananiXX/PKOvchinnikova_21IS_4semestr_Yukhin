# Бизнес-требования к системе управления проектами

## 1. Введение

### 1.1 Назначение документа
Документ определяет бизнес-требования к системе управления проектами с аналитикой и отчетностью. Цель системы - автоматизация процессов управления учебными и исследовательскими проектами в образовательном учреждении.

### 1.2 Область действия
Система предназначена для использования в учебных заведениях (университетах, колледжах) для управления студенческими проектами, курсовыми и дипломными работами.

### 1.3 Цели и задачи
**Основные цели:**
- Повышение эффективности управления проектами на 30%
- Сокращение времени на формирование отчетности на 70%
- Улучшение качества аналитики по проектным показателям

**Задачи:**
- Централизованное хранение информации о проектах
- Автоматизация процессов отслеживания статусов
- Генерация стандартизированных отчетов
- Анализ эффективности проектной деятельности

## 2. Бизнес-контекст

### 2.1 Актуальная ситуация
**Проблемы текущего процесса:**
1. Информация о проектах хранится разрозненно (Excel, Word, бумажные носители)
2. Отсутствие единой системы учета технологий и инструментов
3. Ручное формирование отчетов занимает 3-5 часов в неделю
4. Невозможность оперативного получения аналитики
5. Ошибки при передаче информации между преподавателями

**Экономические потери:**
- Потери времени: 15-20 часов в месяц на административные задачи
- Стоимость ошибок: до 5% проектов теряются из-за плохого учета
- Задержки в отчетности: штрафы за несвоевременную отчетность

### 2.2 Целевая аудитория
| Роль | Количество | Основные задачи |
|------|------------|-----------------|
| Преподаватель | 20-50 чел. | Руководство проектами, оценка, отчетность |
| Студент | 200-500 чел. | Работа над проектами, сдача результатов |
| Методист | 2-5 чел. | Аналитика, формирование отчетов для руководства |
| Руководство кафедры | 3-10 чел. | Контроль, принятие решений на основе аналитики |

## 3. Бизнес-требования

### 3.1 BR-001: Централизованное управление проектами
**Описание:** Система должна предоставлять единую платформу для хранения и управления всей информацией о проектах.

**Бизнес-ценность:**
- Сокращение времени поиска информации на 80%
- Устранение дублирования данных
- Повышение прозрачности процессов

**Критерии успеха:**
- 100% проектов кафедры в системе
- Сокращение времени на поиск информации до 2 минут
- Полное исключение бумажных носителей

### 3.2 BR-002: Автоматизированная отчетность
**Описание:** Система должна автоматически генерировать отчеты в стандартных форматах (Excel, Word).

**Бизнес-ценность:**
- Сокращение трудозатрат на формирование отчетов
- Стандартизация форматов отчетности
- Минимизация человеческих ошибок

**Критерии успеха:**
- Время генерации отчета: < 2 минут
- Ошибки в отчетах: 0%
- Соответствие стандартам учебного заведения

### 3.3 BR-003: Аналитика проектной деятельности
**Описание:** Система должна предоставлять аналитические отчеты по ключевым показателям проектов.

**Бизнес-ценность:**
- Возможность принятия обоснованных решений
- Выявление тенденций и проблем
- Оптимизация распределения ресурсов

**Критерии успеха:**
- Наличие 5+ ключевых метрик
- Время получения аналитики: < 5 минут
- Точность данных: 99.9%

### 3.4 BR-004: Учет используемых технологий
**Описание:** Система должна отслеживать технологии и инструменты, используемые в проектах.

**Бизнес-ценность:**
- Понимание технологического стека
- Планирование обучения
- Соответствие требованиям рынка труда

**Критерии успеха:**
- Учет 100% используемых технологий
- Анализ популярности технологий
- Интеграция с учебными планами

## 6. Ограничения и допущения

### 6.1 Бизнес-ограничения
1. **Бюджет:** Разработка в рамках учебного проекта, без коммерческого бюджета
2. **Сроки:** Разработка и внедрение - 3 месяца
3. **Персонал:** Обучение пользователей силами разработчиков
4. **Инфраструктура:** Использование существующих серверов учебного заведения

### 6.2 Технические ограничения
1. Использование бесплатного ПО и библиотек с открытым исходным кодом
2. Поддержка только локальной установки (on-premise)
3. Отсутствие мобильной версии на первом этапе
4. Лимит на размер генерируемых отчетов - 100 МБ

### 6.3 Допущения
1. Пользователи имеют базовые навыки работы с ПК
2. Доступ к интернету не обязателен после установки
3. Система администрирования PostgreSQL уже настроена
4. Пользователи согласны на миграцию с существующих систем
