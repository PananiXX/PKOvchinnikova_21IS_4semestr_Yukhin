# План тестирования - Система управления портфолио

## 1. Введение

### 1.1. Цель документа
Настоящий документ описывает план тестирования системы управления портфолио. Цель - обеспечить качественную проверку всех функциональных и нефункциональных требований, выявление дефектов перед внедрением системы.

### 1.2. Область тестирования
Тестированию подлежат все модули системы:
- Модуль работы с базой данных
- Модуль графического интерфейса
- Модуль управления записями
- Модуль управления соавторами
- Модуль экспорта данных
- Модуль аналитики и отчетности

### 1.3. Цели тестирования
- Проверка соответствия системы бизнес-требованиям
- Обеспечение стабильности и надежности работы
- Проверка удобства использования интерфейса
- Верификация корректности обработки данных
- Проверка совместимости с различными ОС

## 2. Подход и стратегия тестирования

### 2.1. Типы тестирования
Применяться будут следующие типы тестирования:

#### 2.1.1. Функциональное тестирование
- **Тестирование по позитивным сценариям** - проверка корректной работы функций
- **Тестирование по негативным сценариям** - проверка обработки ошибок и исключительных ситуаций
- **Тестирование граничных значений** - проверка поведения на границах допустимых значений

#### 2.1.2. Тестирование интерфейса
- **Тестирование пользовательского интерфейса** - проверка удобства использования
- **Тестирование совместимости** - проверка работы на разных ОС
- **Тестирование доступности** - проверка поддержки управления с клавиатуры

#### 2.1.3. Интеграционное тестирование
- **Тестирование взаимодействия модулей** - проверка интеграции между компонентами
- **Тестирование работы с БД** - проверка корректности операций с базой данных
- **Тестирование файловой системы** - проверка операций с файлами

#### 2.1.4. Тестирование производительности
- **Тестирование времени отклика** - проверка производительности операций
- **Тестирование нагрузки** - проверка работы при большом количестве данных

#### 2.1.5. Регрессионное тестирование
- **Повторное тестирование** - проверка, что исправления не сломали существующий функционал

### 2.2. Критерии начала тестирования
1. Завершена разработка всех модулей системы
2. Проведено модульное тестирование разработчиками
3. Создана тестовая среда
4. Заполнена тестовая база данных
5. Утверждены тестовые сценарии

### 2.3. Критерии завершения тестирования
1. Все критические и блокирующие дефекты устранены
2. Все обязательные тестовые сценарии выполнены
3. Уровень покрытия тестами ≥ 85%
4. Все автоматизированные тесты успешно пройдены
5. Получено одобрение от руководителя тестирования

import pytest

#### 5.3.2. Unit-тест для валидации года
```python
def test_year_validation():
    """Тест валидации года"""
    from portfolio_app import PortfolioApp
    
    # Тест граничных значений
    valid_years = [2000, 2024, 2030]
    invalid_years = [1999, 2031, "не число"]
    
    for year in valid_years:
        assert PortfolioApp.validate_year(year) == True
    
    for year in invalid_years:
        assert PortfolioApp.validate_year(year) == False
```

## 10. Метрики тестирования

### 10.1. Ключевые метрики
| Метрика | Формула расчета | Целевое значение |
|---------|-----------------|------------------|
