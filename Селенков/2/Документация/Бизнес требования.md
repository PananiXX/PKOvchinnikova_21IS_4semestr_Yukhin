# Бизнес-требования к системе "Портфолио исследователя"

## 1. Введение

### 1.1 Цель документа
Документ содержит бизнес-требования к разработке системы управления портфолио для студентов и исследователей, предназначенной для отслеживания академических и исследовательских достижений.

### 1.2 Общее описание продукта
Система представляет собой десктопное приложение для управления персональным портфолио исследовательской деятельности, включающее ведение записей о проектах, публикациях, конференциях, практиках и грантах.

### 1.3 Целевая аудитория
- Студенты высших учебных заведений
- Аспиранты и молодые ученые
- Исследователи в начале карьеры
- Научные руководители (для мониторинга прогресса студентов)

## 2. Бизнес-цели

### 2.1 Основные цели
1. **Упрощение документирования** исследовательской деятельности
2. **Автоматизация** отслеживания достижений и прогресса
3. **Визуализация** развития компетенций
4. **Формирование** структурированных отчетов для научных руководителей
5. **Мотивация** студентов к систематической работе над портфолио

### 2.2 Ключевые показатели успеха (KPI)
- Увеличение полноты портфолио студентов на 70%
- Сокращение времени на формирование отчетов на 80%
- Увеличение удовлетворенности пользователей на 60%
- Рост количества систематически ведущих портфолио студентов на 50%

## 3. Функциональные требования

### 3.1 Управление записями портфолио
**ID:** BR-F-001
**Описание:** Система должна позволять создавать, редактировать и просматривать записи о различных видах исследовательской деятельности.

**Подтребования:**
- BR-F-001.1: Каждая запись должна содержать обязательные поля: название, тип, дата
- BR-F-001.2: Дополнительные поля: описание, соавторы
- BR-F-001.3: Типы записей: Проект, Публикация, Конференция, Практика, Грант
- BR-F-001.4: Хранение истории изменений не требуется
- BR-F-001.5: Возможность удаления записей

### 3.2 Система ключевых слов
**ID:** BR-F-002
**Описание:** Пользователь должен иметь возможность добавлять ключевые слова к записям для категоризации.

**Подтребования:**
- BR-F-002.1: Ручной ввод до 5 ключевых слов на запись
- BR-F-002.2: Система должна предлагать ранее использованные ключевые слова через автодополнение
- BR-F-002.3: Хранение ключевых слов в отдельной таблице для повторного использования
- BR-F-002.4: Связь многие-ко-многим между записями и ключевыми словами

### 3.3 Исследовательская карта
**ID:** BR-F-003
**Описание:** Визуализация статистики по портфолио в виде текстовой сводки.

**Подтребования:**
- BR-F-003.1: Отображение ключевых слов с количеством связанных записей
- BR-F-003.2: Отображение списка соавторов с количеством совместных работ
- BR-F-003.3: Автоматическое обновление статистики при добавлении новых записей
- BR-F-003.4: Сортировка по частоте использования/количеству работ

### 3.4 Система достижений
**ID:** BR-F-004
**Описание:** Геймификация процесса ведения портфолио через систему достижений.

**Подтребования:**
- BR-F-004.1: Автоматическая разблокировка достижений по заданным правилам
- BR-F-004.2: Отдельная вкладка для просмотра полученных достижений
- BR-F-004.3: Набор достижений:
  - "Первый шаг" — первая запись
  - "Командный игрок" — 3+ записей с соавторами
  - "Разносторонний" — записи 3+ разных типов
  - "Плодотворный год" — 3+ записи за календарный год
  - "Словобог" — суммарный объем описаний > 5000 символов
- BR-F-004.4: Уведомление пользователя о получении достижения

### 3.5 Трекер компетенций
**ID:** BR-F-005
**Описание:** Оценка и отслеживание развития профессиональных компетенций.

**Подтребования:**
- BR-F-005.1: Выбор специальности из доступных
- BR-F-005.2: Загрузка эталонных компетенций из JSON-файла
- BR-F-005.3: Возможность выбора 1-3 компетенций при добавлении записи
- BR-F-005.4: Оценка уровня компетенции по шкале 1-5
- BR-F-005.5: Хранение истории оценок компетенций

### 3.6 Дашборд компетенций
**ID:** BR-F-006
**Описание:** Визуализация прогресса по компетенциям и рекомендации.

**Подтребования:**
- BR-F-006.1: Отображение среднего уровня по каждой компетенции
- BR-F-006.2: Выделение компетенций с уровнем ниже 3 ("слабые зоны")
- BR-F-006.3: Генерация персонализированных рекомендаций для развития
- BR-F-006.4: Категоризация компетенций (технические, коммуникационные и др.)

### 3.7 Экспорт отчетов
**ID:** BR-F-007
**Описание:** Формирование структурированных отчетов для внешнего использования.

**Подтребования:**
- BR-F-007.1: Экспорт в формате .docx (обязательный)
- BR-F-007.2: Включение полного списка записей
- BR-F-007.3: Включение сводки по ключевым словам и соавторам
- BR-F-007.4: Включение профиля компетенций
- BR-F-007.5: Включение персонализированных рекомендаций
- BR-F-007.6: Включение списка достижений
- BR-F-007.7: Автоматическое именование файлов с датой

### 3.8 Управление целями
**ID:** BR-F-008
**Описание:** Постановка и отслеживание учебных и исследовательских целей.

**Подтребования:**
- BR-F-008.1: Возможность установки целей на семестр
- BR-F-008.2: Типы целей: количество записей, уровень компетенций
- BR-F-008.3: Отображение прогресса в реальном времени
- BR-F-008.4: Индикация выполнения/невыполнения целей
- BR-F-008.5: Возможность удаления целей

## 4. Нефункциональные требования

### 4.1 Технические требования
**ID:** BR-NF-001
**Подтребования:**
- BR-NF-001.1: Интерфейс на основе Tkinter
- BR-NF-001.2: База данных PostgreSQL с резервным вариантом SQLite
- BR-NF-001.3: Русскоязычный интерфейс и названия в БД
- BR-NF-001.4: Отсутствие комментариев в коде (по требованию заказчика)
- BR-NF-001.5: Ограниченный набор библиотек (только tkinter, psycopg2/sqlite3, python-docx, json)

### 4.2 Требования к производительности
**ID:** BR-NF-002
**Подтребования:**
- BR-NF-002.1: Отклик интерфейса < 100 мс
- BR-NF-002.2: Загрузка данных < 2 секунд
- BR-NF-002.3: Экспорт отчета < 5 секунд
- BR-NF-002.4: Поддержка до 1000 записей без деградации производительности

### 4.3 Требования к надежности
**ID:** BR-NF-003
**Подтребования:**
- BR-NF-003.1: Автоматическое сохранение при добавлении записей
- BR-NF-003.2: Валидация вводимых данных
- BR-NF-003.3: Обработка ошибок подключения к БД
- BR-NF-003.4: Целостность данных при удалении записей (каскадное удаление)

### 4.4 Требования к удобству использования
**ID:** BR-NF-004
**Подтребования:**
- BR-NF-004.1: Интуитивно понятный интерфейс
- BR-NF-004.2: Валидация полей ввода
- BR-NF-004.3: Автодополнение для часто используемых полей
- BR-NF-004.4: Обратная связь при действиях пользователя
- BR-NF-004.5: Ясные сообщения об ошибках

### 4.5 Требования к безопасности
**ID:** BR-NF-005
**Подтребования:**
- BR-NF-005.1: Локальное хранение данных
- BR-NF-005.2: Защита от SQL-инъекций через параметризованные запросы
- BR-NF-005.3: Отсутствие конфиденциальных данных в системе

## 5. Бизнес-правила

### 5.1 Правила валидации
**ID:** BR-BR-001
**Описание:** Правила проверки вводимых данных.
- BR-BR-001.1: Дата должна быть в формате ГГГГ-ММ-ДД
- BR-BR-001.2: Не более 5 ключевых слов на запись
- BR-BR-001.3: Не более 3 компетенций на запись
- BR-BR-001.4: Уровень компетенции от 1 до 5

### 5.2 Правила разблокировки достижений
**ID:** BR-BR-002
**Описание:** Условия получения достижений.
- BR-BR-002.1: Достижения проверяются при каждой новой записи
- BR-BR-002.2: Повторное получение достижений невозможно
- BR-BR-002.3: Уведомление пользователя при получении

### 5.3 Правила расчета статистики
**ID:** BR-BR-003
**Описание:** Алгоритмы расчета показателей.
- BR-BR-003.1: Средний уровень компетенции = среднее арифметическое всех оценок
- BR-BR-003.2: Слабая зона = уровень компетенции < 3
- BR-BR-003.3: Соавторы разбираются по запятой

## 6. Сценарии использования

### 6.1 Основные сценарии
1. **Добавление новой записи в портфолио**
2. **Просмотр статистики и исследовательской карты**
3. **Анализ развития компетенций**
4. **Формирование отчета для научного руководителя**
5. **Постановка и отслеживание целей на семестр**

### 6.2 Типичный пользовательский сценарий
**Пользователь:** Студент 3 курса
**Цель:** Подготовить отчет о научной деятельности за семестр
**Шаги:**
1. Открывает приложение
2. Проверяет список добавленных записей
3. Анализирует статистику во вкладке "Исследовательская карта"
4. Изучает рекомендации по развитию компетенций
5. Нажимает "Экспорт в Word"
6. Получает готовый отчет для предоставления научному руководителю

## 7. Ограничения

### 7.1 Технические ограничения
- Только десктопная версия (нет веб- или мобильной версии)
- Нет многопользовательского режима
- Нет синхронизации между устройствами
- Нет API для интеграции с внешними системами

### 7.2 Бизнес-ограничения
- Система предназначена только для индивидуального использования
- Нет ролевой модели (только один тип пользователя)
- Нет системы уведомлений или напоминаний
- Отсутствие аналитики для администраторов
