# Глоссарий терминов мессенджера

## А

### Аккаунт (Account)
Учетная запись пользователя в системе мессенджера, содержащая идентификационные данные, настройки и историю активности.

### API (Application Programming Interface)
Набор методов и протоколов для взаимодействия между клиентскими приложениями и сервером мессенджера.

### Асинхронное сообщение
Сообщение, которое может быть отправлено и получено в разное время, без необходимости одновременного присутствия отправителя и получателя онлайн.

### Аутентификация (Authentication)
Процесс проверки подлинности пользователя при входе в систему (обычно через логин и пароль).

### Авторизация (Authorization)
Процесс предоставления пользователю прав на выполнение определенных действий в системе.

## Б

### БД (База данных)
Структурированный набор данных, используемый для хранения информации о пользователях, сообщениях, чатах и другой системной информации.

### Бэкенд (Backend)
Серверная часть приложения, отвечающая за обработку запросов, бизнес-логику и взаимодействие с базой данных.

## В

### Веб-сокет (WebSocket)
Протокол для установления двусторонней связи между клиентом и сервером в реальном времени.

### Входящие сообщения
Сообщения, полученные пользователем от других участников чата.

## Г

### Групповой чат (Group Chat)
Чат, в котором участвуют три и более пользователя.

### Голосовое сообщение
Аудиосообщение, записанное пользователем и отправленное в чат.

## Д

### Девайс (Device)
Физическое устройство (смартфон, планшет, компьютер), на котором установлено клиентское приложение мессенджера.

### Дельта-обновления (Delta Updates)
Технология отправки только изменений в данных вместо полной синхронизации, что уменьшает трафик.

### Доставка сообщения
Статус, подтверждающий, что сообщение успешно доставлено на сервер.

## И

### Идентификатор пользователя (User ID)
Уникальный номер, присваиваемый каждому пользователю при регистрации в системе.

### Исходящие сообщения
Сообщения, отправленные пользователем другим участникам чата.

## К

### Клиент (Client)
Приложение, установленное на устройстве пользователя для взаимодействия с мессенджером.

### Кэш (Cache)
Временное хранилище данных на устройстве пользователя для ускорения работы приложения и работы в офлайн-режиме.

## М

### Медиаконтент
Фотографии, видео, аудиофайлы и другие мультимедийные материалы, отправляемые в чатах.

### Мессенджер (Messenger)
Приложение для обмена сообщениями в реальном времени между пользователями.

## О

### Офлайн-режим
Работа приложения без активного подключения к интернету, с возможностью отправки сообщений в очередь.

### Онлайн-статус
Индикатор, показывающий, активен ли пользователь в данный момент в приложении.

## П

### Пользователь (User)
Человек, использующий мессенджер для общения.

### Протокол
Набор правил и стандартов, определяющих формат обмена данными между компонентами системы.

### Пуш-уведомление (Push Notification)
Уведомление, отправляемое на устройство пользователя, когда приложение не активно.

## Р

### Реальное время (Real-time)
Немедленная обработка и доставка сообщений без заметной задержки.

### Регистрация
Процесс создания новой учетной записи пользователя в системе.

## С

### Сервер (Server)
Удаленный компьютер или кластер компьютеров, обрабатывающий запросы от клиентов и хранящий данные.

### Сессия
Период активности пользователя от входа в систему до выхода.

### Синхронизация
Процесс обновления данных на клиентском устройстве в соответствии с данными на сервере.

### Сквозное шифрование (End-to-End Encryption)
Метод шифрования, при котором сообщения шифруются на устройстве отправителя и расшифровываются только на устройстве получателя.

### Сообщение (Message)
Единица обмена информацией между пользователями, может содержать текст, медиа или другие типы контента.

### Статус прочтения (Read Receipt)
Индикатор, показывающий, что получатель просмотрел сообщение.

## Т

### Токен (Token)
Уникальный ключ, используемый для аутентификации пользователя без необходимости каждый раз вводить пароль.

### Тред (Thread)
Цепочка сообщений, обычно в ответ на конкретное сообщение.

## Ф

### Фронтенд (Frontend)
Клиентская часть приложения, отвечающая за пользовательский интерфейс и взаимодействие с пользователем.

## Ч

### Чат (Chat)
Виртуальное пространство для обмена сообщениями между двумя или более пользователями.

### Чат-бот (Chatbot)
Автоматизированная программа, способная общаться с пользователями через интерфейс мессенджера.

## Ш

### Шифрование (Encryption)
Процесс преобразования информации в форму, недоступную для прочтения без специального ключа.

## Э

### Эмодзи (Emoji)
Графические символы (смайлики), используемые для выражения эмоций в текстовых сообщениях.

## Технические термины

### HTTP (HyperText Transfer Protocol)
Протокол передачи данных, используемый для общения между клиентом и сервером.

### HTTPS (HTTP Secure)
Безопасная версия HTTP с шифрованием данных.

### JWT (JSON Web Token)
Стандарт для создания токенов доступа на основе JSON.

### JSON (JavaScript Object Notation)
Формат обмена данными, используемый для передачи структурированной информации.

### REST (Representational State Transfer)
Архитектурный стиль для построения веб-сервисов.

### SQL (Structured Query Language)
Язык программирования для работы с реляционными базами данных.

### SSL/TLS (Secure Sockets Layer/Transport Layer Security)
Протоколы шифрования для безопасного соединения.

### UUID (Universally Unique Identifier)
Уникальный идентификатор, используемый для однозначной идентификации объектов в системе.

### WebRTC (Web Real-Time Communication)
Технология для организации аудио- и видеосвязи через браузер.

### XMPP (Extensible Messaging and Presence Protocol)
Открытый протокол для обмена сообщениями и информацией о присутствии.

## Статусы сообщений

### Отправляется (Sending)
Сообщение находится в процессе отправки на сервер.

### Отправлено (Sent)
Сообщение успешно доставлено на сервер.

### Доставлено (Delivered)
Сообщение доставлено на устройство получателя.

### Прочитано (Read)
Получатель открыл и просмотрел сообщение.

### Не доставлено (Failed)
Сообщение не удалось доставить из-за ошибки.

---

## Типы чатов

### Личный чат (Private Chat)
Чат между двумя пользователями.

### Групповой чат (Group Chat)
Чат с несколькими участниками.

### Канал (Channel)
Чат с односторонней связью, где администратор публикует сообщения, а подписчики только читают.

### Тематический чат (Topic Chat)
Групповой чат, организованный вокруг определенной темы.

---

## Форматы сообщений

### Текстовое сообщение
Сообщение, содержащее только текст.

### Медиа-сообщение
Сообщение, содержащее изображение, видео или аудио.

### Геолокационное сообщение
Сообщение с прикрепленной геопозицией.

### Контактное сообщение
Сообщение с информацией о контакте.

### Файловое сообщение
Сообщение с прикрепленным файлом.

### Голосовое сообщение
Сообщение в формате аудиозаписи.
