# **ПЛАН УЧЕБНОЙ ПРАКТИКИ (УП.01)**
для обучающихся по специальности 09.02.07 «Информационные системы и программирование»
2 курс, 4 семестр
(в рамках освоения профессионального модуля ПМ.01 «Разработка программных модулей»)

#### 1. Вид практики: Учебная практика.
#### 2. Способ проведения: Рассредоточенная, на базе учебных лабораторий колледжа.
#### 3. Основание: Рабочая программа профессионального модуля ПМ.01 и учебный план специальности.

#### 4. Цели практики:
*   Сформировать первоначальный практический опыт разработки полнофункционального веб-приложения.
*   Закрепить навыки работы с современным стеком технологий: Python (FastAPI), PostgreSQL, Nest.JS/React и Telegram Bot API.
*   Развить общие и профессиональные компетенции в области анализа задач, проектирования, программирования, тестирования и документирования.

#### 5. Задачи практики:
*   Проанализировать техническое задание на разработку информационной системы.
*   Спроектировать архитектуру приложения «клиент-сервер-база данных».
*   Реализовать backend-API на Python с использованием FastAPI и интеграцией с PostgreSQL.
*   Разработать клиентское приложение на Nest.JS или React либо Telegram-бота для взаимодействия с API.
*   Протестировать взаимодействие всех компонентов системы.
*   Оформить проектную документацию и исходный код в Git-репозитории.
*   Подготовить отчет и защитить результаты работы.

#### 6. Место и сроки проведения:
*   Место: Учебные лаборатории информационных технологий колледжа.
*   Срок: 4 недели в соответствии с календарным учебным графиком.
*   Трудоемкость: 144 часа (36 часов в неделю).

#### 7. Тема итогового проекта:
Разработка прототипа информационной системы «Система оценивания работ» с использованием стека технологий Python (FastAPI), PostgreSQL, Nest.JS/React и Telegram Bot.

#### 8. Содержание работы и календарный график:

##### Неделя 1. Проектирование и подготовка инфраструктуры.
*   Содержание работы:
    1.  Анализ технического задания. Определение функциональных требований к системе.
    2.  Разработка схемы взаимодействия компонентов (API, БД, Frontend, Bot).
    3.  Проектирование логической модели базы данных. Создание ER-диаграммы.
    4.  Настройка рабочего окружения: установка и настройка IDE VSCode, СУБД PostgreSQL, Python, Node.js.
    5.  Создание и настройка Git-репозитория на GitHub. Написание начального .gitignore.
*   Контрольный результат недели:
    *   Утвержденная схема архитектуры приложения.
    *   ER-диаграмма базы данных и SQL-скрипт для создания таблиц (init_db.sql).
    *   Созданный и доступный репозиторий на GitHub.

##### Неделя 2. Разработка серверной части (Backend API).
*   Содержание работы:
    1.  Инициализация FastAPI-приложения. Настройка структуры проекта.
    2.  Подключение к базе данных PostgreSQL с использованием асинхронных драйверов (asyncpg) и ORM (SQLAlchemy).
    3.  Создание Pydantic-моделей для валидации данных и SQLAlchemy-моделей для работы с БД.
    4.  Разработка CRUD-эндпоинтов для основной сущности «Задача» (создание, чтение, обновление, удаление).
    5.  Добавление базовой маршрутизации и обработки ошибок.
    6.  Настройка CORS для обеспечения взаимодействия с будущим frontend.
*   Контрольный результат недели:
    *   Рабочее FastAPI-приложение, запускаемое локально.
    *   Коллекция проверенных запросов к API в Postman или Insomnia.
    *   Реализованные основные эндпоинты, работающие с тестовой базой данных.

##### Неделя 3. Разработка клиентской части (Frontend / Telegram Bot).
*   Содержание работы (на выбор студента или по указанию руководителя):
    *   Вариант A (Веб-интерфейс на Nest.JS/React):
        1.  Создание проекта на выбранном фреймворке.
        2.  Разработка пользовательского интерфейса для отображения списка задач и формы создания/редактирования.3.  Реализация логики для отправки HTTP-запросов (fetch/axios) к разработанному API.
        4.  Обеспечение отображения данных,полученных от сервера.
    *   Вариант Б (Telegram Bot):
        1.  Создание нового бота через @id199142634 (@BotFather) и получение токена.
        2.  Разработка бота на библиотеке aiogram (асинхронной).
        3.  Реализация команд (например, /start, /tasks, /new) и обработка callback-запросов от кнопок.
        4.  Интеграция бота с ранее созданным API: отправка запросов для получения списка задач, создания новой задачи.
*   Контрольный результат недели:
    *   Работающее клиентское приложение (локальный веб-сервер или функционирующий Telegram-бот).
    *   Демонстрация полного цикла: действие в клиенте -> запрос к API -> изменение в БД -> ответ клиенту.

##### Неделя 4. Интеграция, тестирование, защита.
*   Содержание работы:
    1.  Комплексная проверка работы всей системы. Сквозное тестирование.
    2.  Отладка и исправление выявленных ошибок. Рефакторинг кода.
    3.  Написание ключевых модульных тестов для API (с использованием pytest).
    4.  Оформление проекта: создание подробного README.md с описанием, инструкцией по установке и запуску.
    5.  Подготовка итоговой презентации (5-7 слайдов) и демонстрационного сценария.
    6.  Написание и оформление отчета по практике.
    7.  Защита проекта перед аттестационной комиссией.
*   Контрольный результат недели:
    *   Стабильно работающий прототип информационной системы.
    *   Заполненный Git-репозиторий с качественным кодом и документацией.
    *   Завершенный письменный отчет по практике.
    *   Презентация и успешная устная защита.

#### 9. Формы отчетности:
1.  Отчет по учебной практике в печатном виде, содержащий все структурные элементы.
2.  Программный продукт: ссылка на публичный Git-репозиторий (GitHub/GitLab) с исходным кодом и историей коммитов.
3.  Презентация для защиты в формате PPTX или PDF.
4.  Устный доклад и демонстрация работы приложения на защите.

#### 10. Критерии оценки результатов:
*   Качество реализации проекта (40%): функциональная полнота, работоспособность, корректность интеграции компонентов.
*   Техническое качество кода (30%): чистота, структурированность, наличие комментариев, логичность истории Git.
*   Качество документации и отчета (20%): полнота, грамотность, четкость описания.
*   Защита проекта (10%): качество доклада, демонстрации, ответы на вопросы.
